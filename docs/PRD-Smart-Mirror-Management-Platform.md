# Smart Mirror Management Platform — 产品需求文档

> v4.0 · 2026-03-26

---

## 1. 产品概述

智能试衣镜管理平台（Smart Mirror Management Platform）是一套面向 **平台运营方、渠道商、终端客户** 三类角色的多租户 SaaS 管理系统。平台负责账户体系与激活码的总调度；渠道商作为中间层将资源分发给其下属客户；终端客户管理自有服装库、模板及设备，驱动智能试衣镜完成消费者互动。

---

## 2. 用户角色

| 角色 | Portal | 说明 |
|------|--------|------|
| 平台超管（Platform Super Admin） | Platform Portal | 系统最高权限，管理全局渠道、客户、资产与激活码 |
| 渠道管理员（Channel Owner） | Channel Portal | 管理本渠道下属客户，持有激活码池并分发 |
| 客户管理员（Customer Admin / HQ Owner） | Customer Portal | 管理本组织服装库、模板、设备及成员 |

**登录后跳转规则**

| 角色 | 默认落地页 |
|------|-----------|
| 平台超管 | /dashboard/channels（渠道列表） |
| 渠道管理员 | /dashboard/channel/customers（客户列表） |
| 客户管理员 | /dashboard/customer/users（成员管理） |

---

## 3. 权限控制

权限执行三层：**导航菜单隐藏 → 路由拦截（403 页面） → 按钮级 PermGuard（置灰或隐藏）**

- 未登录访问任何内页 → 重定向登录页
- 已登录但无权限访问某路由 → 跳转 403 页面
- 组织被禁用时，该组织下的账号登录 → 提示"账号所属组织已停用"

## 4. 激活码机制

### 4.1 激活码类型

| 类型 | 说明 |
|------|------|
| Regular（正式码） | 无使用次数上限，设备永久绑定 |
| Trial（试用码） | 含最大试穿次数配额（25 次或 50 次），次数耗尽后状态变为 Expired |

### 4.2 激活码状态流转

```
Unused ──── 设备绑定 ──── Bound
  │                         │
  │                    (Trial 次数耗尽)
  │                         ↓
  │                      Expired
  │
  └──── 管理员撤销 ──── Revoked
```

- **Unused**：已创建，未绑定任何设备
- **Bound**：已绑定设备，正常使用中
- **Expired**：Trial 码次数耗尽（Regular 码不会 Expired）
- **Revoked**：被管理员手动撤销，不可恢复

仅 **Unused** 状态的激活码可被撤销（Revoke）。

### 4.3 两条分发路径

**路径 A：渠道路径（Channel Path）**

```
平台超管
  └→ 创建 Regular 码，批量分配到渠道码池
       └→ 渠道管理员 查看/管理 渠道码池
            └→ 从渠道码池中指定一个 Regular 码，分配给某客户
                 或 渠道管理员自主创建 Trial 码 分配给某客户
```

**路径 B：直销路径（Direct Path）**

```
平台超管
  └→ 直接为某直签客户创建 Regular 码 或 Trial 码
```

### 4.4 客户激活码上限

- 每个客户账号的有效激活码总数（**Regular + Trial，Bound + Unused 合计**）上限为 **10 个**
- Expired / Revoked 状态的码不计入上限
- Trial 码专项上限：同一客户下 Trial 码（Bound + Unused）合计不超过 **10 个**（含在总上限内）
- 达到上限后，Create Trial Code 按钮置灰；渠道 Assign Regular Codes 按钮仍可操作直至总上限

### 4.5 激活码与设备绑定

- 每个激活码为**一次性**使用，绑定后不可迁移到其他设备
- 设备通过在设备端输入激活码完成绑定，绑定后状态变为 Bound
- 绑定后，客户管理员可为该设备（激活码记录）设置或修改**昵称**（最长 50 字符）

## 5. 平台 Portal 功能

### 5.1 渠道管理

**功能入口**：Channels（左侧导航）

**列表字段**：渠道名称、Channel ID、状态、管理员邮箱、客户数量、创建时间

**操作**

| 操作 | 说明 |
|------|------|
| 创建渠道 | 填写渠道名称（最长 50 字符）+ 管理员邮箱，系统发送邀请邮件 |
| 搜索 | 按渠道名称模糊搜索 |
| 查看客户 | 跳转到该渠道下的客户列表 |
| 管理激活码 | 打开渠道激活码池 Modal |
| 禁用 / 启用 | 禁用后该渠道管理员立即失去访问权限 |

**渠道激活码池 Modal**

- 展示该渠道当前全部码（含已分配给客户的记录）
- 操作：创建 Regular 码（含数量选择，1 至 100 个，批量创建）；Revoke Unused 码
- 码列表字段：激活码、类型、状态、创建时间

**表单校验**

- 渠道名称为空 → 阻止提交
- 渠道名称重复 → 提示"渠道名称已存在"
- 管理员邮箱已注册 → 提示"邮箱已被注册"

---

### 5.2 客户管理（超管视图）

**功能入口**：Customers（左侧导航）

**列表字段**：客户名称、管理员邮箱、状态、所属渠道、激活码数量、创建时间

**操作**

| 操作 | 说明 |
|------|------|
| 创建直签客户 | 填写客户名称（最长 50 字符）+ 管理员邮箱 |
| 管理激活码 | 打开客户激活码 Modal |
| 禁用 / 启用客户 | 禁用后该客户所有用户失去访问权限 |

**客户激活码 Modal**

- 展示该客户全部激活码
- 操作：创建 Regular 码（含数量选择，1 至 100 个）；创建 Trial 码（选择最大试穿次数：25 或 50）；Revoke Unused 码
- 客户处于 Disabled 状态时，创建码按钮置灰
- 码列表字段：激活码、类型、状态、绑定设备 ID、昵称、创建来源（Platform / Channel）、创建时间

---

### 5.3 服装库（超管聚合视图）

**功能入口**：Assets > Garments（左侧导航）

- 展示全系统所有客户的服装，可按客户（仅限 Customer 类型组织）筛选
- 操作：编辑服装信息、删除服装（需二次确认）
- 删除操作：Modal 显示所属组织名称，确认后删除

---

### 5.4 模板库（超管视图）

**功能入口**：Assets > Templates（左侧导航）

**操作**

| 操作 | 说明 |
|------|------|
| 创建模板 | 填写模板名称（最长 100 字符）+ 添加 Prompt（每条最长 200 字符） |
| 编辑模板 | 修改名称、Prompt 列表 |
| 删除模板 | 二次确认；若有分配记录则提示受影响客户数 |
| 分配给客户 | 将模板分配给指定客户 |

**Prompt 规则**：同一模板内不可添加重复 Prompt

---

### 5.5 激活码总览

**功能入口**：Assets > Devices（左侧导航）

- 只读视图，聚合展示全系统所有激活码
- 筛选：按客户、按状态
- 展示字段：激活码、归属客户、状态、绑定设备 ID、昵称、创建来源、创建时间

## 6. 渠道 Portal 功能

### 6.1 客户管理

**功能入口**：Customers（左侧导航）

**列表字段**：客户名称、管理员邮箱、状态、创建时间

**操作**

| 操作 | 说明 |
|------|------|
| 创建渠道客户 | 填写客户名称（最长 50 字符）+ 管理员邮箱，发送邀请 |
| 管理激活码 | 打开该客户的激活码 Modal |
| 重发邀请 | 客户管理员从未登录时显示，重新发送邀请邮件 |
| 禁用 / 启用客户 | 禁用后该客户所有用户失去访问权限 |

**客户激活码 Modal**

- 展示该客户全部激活码
- 操作：
  - **Assign Regular Codes**：点击后打开第二层 Modal，展示渠道码池中可用的 Regular Unused 码列表，每行可逐一点击 Assign 进行分配
  - **Create Trial Code**：选择最大试穿次数（25 或 50），直接在本渠道下创建 Trial 码并分配给该客户
  - Revoke：撤销 Unused 状态的码
- 试用码计数：显示该客户 Trial 码（Bound + Unused）数量及上限（10 个）
- 客户处于 Disabled 状态时，分配/创建码按钮置灰
- 渠道码池无可用 Regular 码时，Assign Regular Codes 按钮置灰

---

### 6.2 激活码池

**功能入口**：Code Pool（左侧导航）

- 展示当前渠道内未分配给客户的全部激活码（`org_id = null`）
- 筛选：按类型（Regular / Trial）、按状态
- 展示字段：激活码、类型、状态、绑定设备 ID、昵称、创建来源、创建时间

---

### 6.3 模板库（渠道只读视图）

**功能入口**：Templates（左侧导航）

- 展示已分配给本渠道的全部模板
- Prompts 列显示该模板下的 Prompt 数量，点击"View Prompts"可在 Modal 中查看每条 Prompt 完整内容（支持长文本展示，编号排列）
- 渠道管理员无法创建、编辑或删除模板

### 7.1 成员管理

**功能入口**：Users（左侧导航）

**操作**：邀请成员（填写邮箱 + 选择角色）、禁用 / 启用成员、重置成员密码

---

### 7.2 服装库

**功能入口**：Garments（左侧导航）

**操作**

| 操作 | 说明 |
|------|------|
| 上传服装 | 上传图片文件（JPEG / PNG / WebP） |
| 编辑服装 | 修改服装名称（最长 100 字符）；图片区域显示当前图片（只读预览），可点击"Upload New"上传本地新图片替换 |
| 删除服装 | 二次确认 |

| 分配到设备 | 从已绑定的设备（激活码 Bound 状态）列表中选择目标设备，支持多选；展示字段：设备 ID、昵称、激活码、类型 |

**异常状态**

- 图片加载失败 → 显示占位图
- 无已绑定设备时 → Assign Devices Modal 显示空状态"暂无已激活设备，请先在 Devices 页面绑定设备"

---

### 7.3 模板库（客户只读视图）

**功能入口**：Templates（左侧导航）

- 展示平台分配给本客户的全部模板，只读
- Prompt 数量点击 → Modal 内展示完整 Prompt 列表

---

### 7.4 设备管理

**功能入口**：Devices（左侧导航）

- 展示本客户全部激活码，客户视角即"设备管理"
- 展示字段：激活码、类型、状态、绑定设备 ID、昵称、创建来源、创建时间
- 操作：
  - Unused 状态：**Bind Device** — 输入设备 ID 完成绑定
  - Bound 状态：**Set / Edit Nickname** — 设置或修改设备昵称（最长 50 字符）
- 激活码由账号管理员（平台超管或渠道管理员）创建并分配，客户端不可自行创建或撤销

## 8. 通用功能

### 8.1 个人设置

**功能入口**：Settings（左侧导航，所有角色可见）

| 字段 | 规则 |
|------|------|
| Display Name | 最长 50 字符 |
| 修改密码 | 先发送邮件验证码（6位数字）→ 输入新密码（≥8字符）+ 确认 |

---

### 8.2 登录 / 忘记密码

**登录**：邮箱 + 密码；邮箱格式校验；凭据错误统一提示（不区分是邮箱错还是密码错）；账号被禁用单独提示。

**忘记密码**：输入邮箱 → 发送验证码 → 输入验证码通过 → 设置新密码 + 确认（两次需一致）。

## 10. 核心业务流程

### 10.1 渠道激活码完整链路

```
1. 平台超管 → 创建若干 Regular 码 → 分配到渠道 A 的码池
2. 渠道 A 管理员 → 进入 Customers → 点击某客户"Manage Codes"
3. 在弹窗中点击"Assign Regular Codes" → 打开码池列表 → 选择一个码点击 Assign
4. 该码 org_id 更新为该客户，出现在客户激活码列表中
5. 渠道 A 管理员 → 也可为该客户"Create Trial Code"（选 25 或 50 次）
6. 客户管理员 → 在 Devices 页面 → 找到 Unused 码 → 点击 Bind Device → 输入设备 ID
7. 码状态变为 Bound，设备正式激活
8. 客户管理员 → 可为 Bound 设备设置昵称
9. 客户管理员 → 在 Garments 页面 → 为服装指定分配设备（从 Bound 设备列表中选）
```

### 10.2 直销客户激活码链路

```
1. 平台超管 → 进入 Customers → 找到直签客户 → 点击"Manage Codes"
2. 创建 Regular 码（含数量选择）或 Trial 码（选次数配额）
3. 后续流程同上（步骤 6 起）
```

### 10.3 客户激活码上限控制

- 判断条件：该客户下所有码中，`status ∈ {Bound, Unused}` 的总数 ≥ 10
- 达上限后：Create Trial Code 置灰，提示已达上限
- Trial 码（Bound + Unused）独立计数，同样以 10 为上限

## 11. 异常与边界状态

| 场景 | 处理规则 |
|------|---------|
| 渠道被禁用 | 该渠道管理员登录提示组织停用；渠道下所有客户不受影响 |
| 客户被禁用 | 该客户所有成员登录提示组织停用；激活码记录保留 |
| 渠道码池为空 | Assign Regular Codes 按钮置灰，提示无可用码 |
| 客户激活码达上限 | Create Trial Code 置灰，展示上限提示 |
| Bound 码不可撤销 | Revoke 按钮仅对 Unused 码显示 |
| 无绑定设备 | Garments Assign Devices Modal 显示空状态引导 |
| 无已分配模板 | 客户模板列表展示空状态 |
| 图片加载失败 | 显示占位图，不报错 |
