# Forgot Password Page

**路径**: `/forgot-password`
**适用门户**: 全部（Platform / Channel / Customer）
**访问权限**: 公开页面，无需登录

---

## 一、页面概述

忘记密码页允许用户通过邮箱验证码完成密码重置。流程分为三步：邮箱验证 → 设置新密码 → 成功提示。

---

## 二、重置密码流程

### Step 1：邮箱验证

| 字段 | 类型 | 说明 |
|------|------|------|
| Email address | 文本输入 | 输入需重置密码的账号邮箱 |
| Verification code | 文本输入 | 用户点击 Send Code 后出现 |

**操作流程：**
1. 用户输入邮箱后点击 **Send Code**，系统向该邮箱发送 6 位数字验证码。
2. 验证码输入框在点击 Send Code 后显示。
3. 用户可点击 **Resend** 重新发送验证码。
4. 用户输入验证码后点击 **Verify & Continue**，通过后进入 Step 2。
5. **Verify & Continue** 按钮在未发送验证码前处于禁用状态（`disabled`）。

**验证码校验规则：**

| 规则 | 说明 |
|------|------|
| 必填 | 验证码为必填项 |
| 长度 | 必须为 6 位 |
| 格式 | 必须为纯数字 |

### Step 2：设置新密码

| 字段 | 类型 | 校验规则 |
|------|------|---------|
| New password | 密码输入 | 必填；至少 8 个字符 |
| Confirm new password | 密码输入 | 必填；必须与新密码一致 |

- 两次输入不一致时，提示："Passwords do not match"。
- 提交后进入 Step 3。

### Step 3：重置成功

页面展示成功提示，提供 **Back to Sign in** 按钮跳转至 `/login`。

---

## 三、步骤进度指示

页面顶部展示步骤进度条（Steps），包含两步：
1. Verify Email
2. New Password

成功页面不显示步骤进度条。

---

## 四、黑盒规则

### R-FP-01：密码最小长度为 8 位
新密码必须不少于 8 个字符，短于此长度的提交会被客户端拦截并提示错误。

### R-FP-02：验证码发送前不可提交
"Verify & Continue" 按钮在用户未触发 Send Code 之前处于禁用状态，防止未完成验证码步骤直接提交。

### R-FP-03：重置密码仅凭邮箱验证完成
密码重置流程不需要输入旧密码，仅通过邮箱验证码完成身份验证后即可设置新密码。

---

## 五、导航

- 在 Step 1 和 Step 2 页面底部展示 **Remember your password? Sign in** 链接，跳转至 `/login`。
- 重置成功后通过 **Back to Sign in** 按钮返回登录页。
